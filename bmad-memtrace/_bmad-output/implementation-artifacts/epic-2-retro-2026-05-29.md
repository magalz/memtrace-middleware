## What Went Well

1. **Fusion Engine as pure function** — `fuse(FusedInput) → Result<FusedContext>`, zero side effects, most testable component in the pipeline.
2. **Two-Phase Quality Gate Architecture** — Phase 1 (textual, human review) + Phase 2 (mathematical, non-bypassable). Well-executed decomposition.
3. **Mandatory, Non-Bypassable Halt** — Blast radius check never silently skipped. Strong architectural commitment.
4. **Ring Buffer O(1) performance** — Backbone for CLI status. 17 unit tests, stress-tested at 100k ops.
5. **Zero external dependencies for gate scripts** — Only Node.js built-ins. Zero-friction setup.
6. **Code review effectiveness** — Caught mutable state, non-functional timeouts, null safety issues, broken patterns.
7. **Rapid re-run on empty reviews** — When code review returned empty, re-execution was painless (noted by Magal).

## Challenges

1. **Timeout implementation correctness** — Most persistent bug pattern. Root cause: opencode desktop IDE instability with local MCP server disconnects (Magal migrated to new IDE at end of Epic 3, resolving the underlying issue).

2. **XML step structure management** — 4 stories modifying same `SKILL.md`, adding substeps inside step 5. Chain grew to 4+ quality gates in single step.

3. **Null/undefined edge cases** — Null items in arrays (2.1a), `startsWith()` on undefined (2.3b), null candidates crash (2.4). External input defensiveness consistently underestimated.

4. **Degradation state machine absent** — Story 2.3a hardcoded `DegradationTier.Full`. CLI status display partially cosmetic until Epic 3 implementation.

5. **Pre-existing technical debt surfacing** — 10 pre-existing issues flagged in 2.1b review (UTF-8 corruption, command injection risk, etc.), all deferred.

## Key Insights

1. **Never trust external API shape** — Even when types say a field exists, null/undefined guards are mandatory on all external input.
2. **Pact/contract tests prevent schema drift bugs** — Should be standard for any new adapter or contract change.
3. **Two-phase quality gates work** — Human-in-the-loop textual review + non-bypassable mathematical gate is a proven pattern.
4. **IDE stability affects test reliability** — MCP server disconnect issues were environmental, not code bugs.
5. **Code review is the safety net that works** — 31 patches caught before merge; zero production issues.
